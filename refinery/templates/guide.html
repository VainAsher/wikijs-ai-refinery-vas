Guide

How the refinery turns raw imported docs into VAS-owned, governed knowledge — and where everything lives.

Open the queue
The one rule: imported third-party content is evidence, not truth. You import → classify → rewrite into an original VAS draft → review → publish. Reference docs are never published verbatim.

The workflow at a glance

1
Import
Bulk Workbench or Connectors pull raw docs into the queue.
2
Classify
Each doc is auto-tagged with source, service, risk & governance.
3
Triage
Filter the queue; bulk-set actions on whole groups.
4
Transform
Rewrite a reference doc into an original VAS draft (AI optional).
5
Review
Edit, approve, or mark canonical in the browser editor.
6
Publish
Push approved docs to Wiki.js (or Export to Markdown).

The safety model — why imports can't become policy by accident
Reference / imported

Source evidence only. Forced non-canonical and not customer-safe. Never published verbatim.

rewrite required
VAS transformed draft

An original, editable VAS-owned working doc generated from the reference. Not auto-published.

draft
Reviewed canonical

Approved source of truth, ready for Wiki.js.

canonical

Governance is deterministic and runs after any AI step, so a competitor/employer/vendor doc cannot be flagged canonical even if a model suggests it.

Recommended first pass for a large import — ~1,500 files
  1. Open Bulk and paste one label|path per source directory. Use limit 0 to import everything. Leave the Ollama box empty for a fast deterministic first pass.
  2. On Docs, filter by source_org, service, and doc_type to find natural groups.
  3. Back in Bulk, set an adaptation action (e.g. rewrite_into_sop) on a whole filtered slice.
  4. Transform the important docs into VAS drafts (add a model like mistral:latest for quality), then review and approve.
  5. Configure Wiki.js in Config and publish, or use Export.
Using local AI (Ollama) — optional, improves rewrite quality

The refinery works fully without AI (deterministic classification + a safe fallback draft template). A local model only improves the quality of AI classification and transforms — it never affects the safety/governance rules.

  • Pick the default model and check it's reachable on Config; use Run test generation to see real words/second on your machine.
  • Larger instruct models (e.g. mistral:latest) produce fuller, more usable drafts. 3B-class models tend to under-generate.
  • Start Ollama with ollama serve; pull a model with ollama pull mistral.

TermMeaning
source_orgWho the doc came from (e.g. employer_hosting, authentik). Drives the governance defaults.
source_roleRelationship to that source: owned, competitor_reference, employer_reference, vendor_documentation, etc.
reuse_policyHow you're allowed to use it: owned_original, rewrite_required, reference_only, quote_prohibited.
adaptation_actionThe transform target (e.g. rewrite_into_sop, rewrite_into_moderation_playbook).
rewrite_statusLifecycle: needs_rewrite → draft_generated → in_review → approved.
authorityTrust level: imported_unreviewed, draft, approved, canonical, archived.
canonicalThis is the VAS source of truth. Only reachable for VAS-owned, reviewed docs.
customer_safeSafe to show clients/community. Off by default and forced off for reference docs and sensitive-scan hits.
context packA Markdown file of VAS knowledge injected into AI rewrites as higher authority than the source.
